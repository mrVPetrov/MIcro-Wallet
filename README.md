# MIcro-Wallet
Проект для тренировки

### Цель: 
Создать надежный API-сервис для управления виртуальными кошельками пользователей, где любые операции с балансом гарантированно атомарны, код проходит автоматические проверки, а процесс разработки имитирует командную работу.

### Бизнес-логика (Что делаем)
1. У пользователя есть один виртуальный кошелек с балансом.
2. Пользователь может:
   - Пополнить баланс (внешнее "зачисление").
   - Перевести средства другому пользователю.
   - Получить историю своих транзакций.
3. Критическое требование: При переводе средств списание у отправителя и зачисление получателю должны происходить одновременно и атомарно. Недопустимы ситуации "деньги списались, но не зачислились" или "гонки данных" (race conditions) при одновременных запросах
